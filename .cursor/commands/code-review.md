

# 코드 품질 리뷰

종합적인 코드 품질 리뷰 수행: $ARGUMENTS

## 현재 상태

- Git 상태: !`git status --porcelain`
- 최근 변경사항: !`git diff --stat HEAD~5`
- 저장소 정보: !`git log --oneline -5`
- 빌드 상태: !`npm run build --dry-run 2>/dev/null || echo "빌드 스크립트 없음"`

## 작업 내용

다음 단계에 따라 철저한 코드 리뷰를 수행하세요:

1. **저장소 분석**
   - 저장소 구조를 검토하고 주요 언어/프레임워크를 식별
   - 구성 파일 확인 (package.json, requirements.txt, Cargo.toml 등)
   - README 및 문서를 검토하여 맥락 파악

2. **코드 품질 평가**
   - 코드 스멜, 안티 패턴, 잠재적 버그를 스캔
   - 일관된 코딩 스타일과 네이밍 규칙 확인
   - 사용하지 않는 import, 변수 또는 데드 코드 식별
   - 오류 처리 및 로깅 관행 검토

3. **보안 검토**
   - 일반적인 보안 취약점 탐색 (SQL 인젝션, XSS 등)
   - 하드코딩된 비밀키, API 키 또는 비밀번호 확인
   - 인증 및 권한 부여 로직 검토
   - 입력 검증 및 정화 확인

4. **성능 분석**
   - 잠재적 성능 병목 지점 식별
   - 비효율적인 알고리즘 또는 데이터베이스 쿼리 확인
   - 메모리 사용 패턴 및 잠재적 누수 검토
   - 번들 크기 및 최적화 기회 분석

5. **아키텍처 및 디자인**
   - 코드 조직화 및 관심사 분리 평가
   - 적절한 추상화 및 모듈성 확인
   - 의존성 관리 및 결합도 검토
   - 확장성 및 유지보수성 평가

6. **테스트 커버리지**
   - 기존 테스트 커버리지 및 품질 확인
   - 적절한 테스트가 부족한 영역 식별
   - 테스트 구조 및 조직화 검토
   - 추가 테스트 시나리오 제안

7. **문서화 검토**
   - 코드 주석 및 인라인 문서 평가
   - API 문서 완성도 확인
   - README 및 설정 지침 검토
   - 더 나은 문서화가 필요한 영역 식별

8. **권장사항**
   - 심각도에 따라 문제 우선순위 지정 (치명적, 높음, 중간, 낮음)
   - 구체적이고 실행 가능한 권장사항 제공
   - 개선을 위한 도구 및 관행 제안
   - 다음 단계가 포함된 요약 보고서 작성

적용 가능한 경우 파일 경로와 라인 번호를 포함한 구체적인 예시를 제공하면서 건설적인 태도를 유지하세요.
